Parkinson's Disease Prediction

Rules for Uploading Data

  • The uploaded file should be in .csv or .data format.
  • The file must contain a column named 'name' with session identifiers (e.g., phon_R01_S01_1).
  • Ensure that all necessary feature columns are included:
    • MDVP:Fo(Hz) - Average vocal fundamental frequency
    • MDVP:Fhi(Hz) - Maximum vocal fundamental frequency
    • MDVP:Jitter(%) - Variation in fundamental frequency
    • ... and other required features
  • The 'status' column is optional, and it will be ignored during prediction.
  • For every prediction, a person will be assigned a value of 1 if they have Parkinson's and 0 if they do not.
  • The model will predict whether the individual has Parkinson's disease based on the uploaded data.

What to Expect in the Output File

  • The output file will contain predictions for each session along with a summary of predictions for each individual.
  • The final predictions will indicate whether an individual is likely to have Parkinson's disease or not.
  • Summary rows will be included in the output file, showing the total number of individuals predicted to have the disease and those who are not.
